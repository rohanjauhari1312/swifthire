# SwiftHire

A Chrome/Brave extension that automates job applications: generate a tailored resume via a Claude.ai project, auto-save it, find a contact's email, and open a pre-filled cold email in Apple Mail with the right resume attached.

> macOS only (uses Apple Mail via AppleScript and a local Python server).

## What's inside

- `job-to-claude/` — the browser extension (Manifest V3).
- `server/swifthire_server.py` — a local HTTP server (`127.0.0.1:9875`) that saves/renames resume PDFs, extracts PDF text, and opens Apple Mail drafts.
- `server/com.swifthire.server.plist` — launchd template to keep the server running.

## Prerequisites

- macOS with Apple Mail set up.
- Python 3 (`/usr/bin/python3`).
- `pdftotext` (for reading resume text): `brew install poppler`.
- A Claude.ai account with a Project that generates your resume from a pasted JD.
- API keys: an [Anthropic API key](https://console.anthropic.com) (email generation) and a [Hunter.io](https://hunter.io) key (email finding).

## Setup

### 1. Configure the extension
Edit `job-to-claude/popup.js`:
- Fill in the `PROFILE` object (name, email, phone, school, etc.).
- Set `PROJECT_URL` to your Claude.ai project URL.

Edit `job-to-claude/background.js` (top config block):
- `PROJECT_URL` — same Claude project URL.
- `DOWNLOAD_DIR` — absolute path where resumes are saved (e.g. `/Users/you/Desktop/Resumes`).
- `RESUME_PREFIX` — resumes are saved as `<prefix>_<company>.pdf` (e.g. `jane_doe`).
- `PROFILE_NAME` / `PROFILE_INTRO` — used in the email subject and opening line.

### 2. Load the extension
1. Go to `brave://extensions` (or `chrome://extensions`).
2. Enable Developer mode.
3. "Load unpacked" → select the `job-to-claude/` folder.

### 3. Run the local server
```
mkdir -p ~/Desktop/Resumes   # or whatever DOWNLOAD_DIR you chose
cp server/com.swifthire.server.plist ~/Library/LaunchAgents/
# edit the path inside the plist to point at server/swifthire_server.py
launchctl load ~/Library/LaunchAgents/com.swifthire.server.plist
```
The first time, macOS will ask Mail/automation permissions — allow them.

### 4. Add your API keys
Open the extension popup → scroll to "API keys" → paste your Anthropic and Hunter keys. They're stored locally in the browser, never in code.

### 5. Browser download setting (recommended)
In `brave://settings/downloads`, turn OFF "Ask where to save each file" so resumes save without a dialog.

## Usage

- **Send to Claude** — on a job posting page: extracts the JD, opens your Claude project, and submits it. Your project generates the resume; it auto-downloads to `DOWNLOAD_DIR` as `<prefix>_<company>.pdf`.
- **Email & Compose** — on a LinkedIn profile: finds the person's company, matches the right JD + resume, finds their email via Hunter, and opens an Apple Mail draft with a tailored cold email and the resume attached. If it can't match confidently, it queues the request — reopen the popup to pick or paste the JD.
- **Fill Form** — on an application form: fills known fields from `PROFILE` and sends open-ended questions to your Claude project; click "Apply Answers" to paste the responses back.

## Notes

- No API keys or personal data are stored in the source. Keys live in `chrome.storage.local`.
- The email is generated by reading the **actual resume PDF text** so bullets reflect the tailored resume, ordered strongest-first.
